===========================================
DESCRIÇÃO GERAL DO PROJETO
===========================================

A solução MindTrack utiliza dois modelos de Inteligência Artificial,
um de Classificação e outro de Regressão, para analisar indicadores de 
saúde mental, estresse ocupacional e produtividade de colaboradores.

Os modelos foram treinados a partir de um dataset real de saúde mental 
no ambiente de trabalho (mental_health_workplace_survey.csv), contendo 
informações relacionadas à satisfação no trabalho, estresse, horas de 
sono, atividade física, apoio do gestor, equilíbrio vida-trabalho, 
produtividade, entre outros fatores.

O objetivo é auxiliar empresas e profissionais a identificar riscos 
potenciais de Burnout e entender como diferentes variáveis influenciam 
a produtividade.

===========================================
MODELO 1 — CLASSIFICAÇÃO (BurnoutRisk)
===========================================

Tipo: Classificação Binária (0 = Sem risco, 1 = Risco de Burnout)

O modelo segue as etapas ensinadas nas apostilas da disciplina, 
incluindo:

- Análise Exploratória
- Separação das variáveis preditivas e alvo
- Treinamento com train_test_split()
- Padronização dos dados (StandardScaler)
- Treinamento do algoritmo (ex.: RandomForestClassifier)
- Avaliação com Matriz de Confusão e Classification Report
- Salvamento do modelo em formato .pkl (modelo_classificacao_burnoutrisk.pkl)

O resultado final permite prever se um colaborador apresenta ou não 
risco de Burnout com base em 23 variáveis comportamentais, sociais e 
profissionais.

===========================================
MODELO 2 — REGRESSÃO (Produtividade)
===========================================

Tipo: Regressão Numérica (score mínimo a máximo de produtividade)

Etapas conforme apostila:

- Seleção das features de regressão
- Engenharia de atributos simples
- Normalização com StandardScaler
- Treinamento com XGBRegressor
- Avaliação com MAE, MSE e R²
- Gráficos Real vs Previsto
- Importância das variáveis
- Salvamento em formato .pkl (modelo_produtividade.pkl)

Este modelo prevê quanto um colaborador deve produzir, considerando 
estresse, sono, carga de trabalho, atividade física, equilíbrio 
vida-trabalho e suporte do gestor.

===========================================
API REST — FLASK
===========================================

Foi desenvolvida uma API Flask com duas rotas:

1. /predict/burnout  → recebe 23 features e retorna 0 ou 1  
2. /predict/produtividade → recebe as variáveis de regressão e retorna o valor previsto

A API carrega automaticamente os modelos .pkl e o StandardScaler.  
A aplicação também possui interface HTML integrada, permitindo testes 
diretos via formulário.
